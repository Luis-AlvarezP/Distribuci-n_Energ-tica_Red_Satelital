// 01 — MODELADO DEL SISTEMA

El Satélite como Servidor Orbital

Un satélite de órbita baja (LEO) es esencialmente un servidor de red autónomo operando a ~550 km de altitud. Su restricción crítica es el Power Budget: la energía eléctrica generada por los paneles solares debe distribuirse entre sus tres subsistemas principales sin saturación ni apagado. El sistema Ax = b modela este balance, donde cada ecuación representa una restricción térmica o energética del bus eléctrico.

📡
x₁
Transmisión de Datos (RF/Láser)
Potencia [W] asignada al enlace de comunicaciones. Conecta el nodo con tierra y otros satélites de la constelación. Alta demanda en downlink.
💻
x₂
Procesamiento de Paquetes (CPU)
Potencia [W] para el router embarcado. Enruta el tráfico IP de usuarios. Genera calor residual que afecta al subsistema térmico del satélite.
⚙️
x₃
Estabilización (Reaction Wheels)
Potencia [W] para los motores de orientación. Mantienen el satélite apuntando correctamente. Consumo variable según maniobras de actitud.
// SIGNIFICADO DE LA MATRIZ A Y VECTOR b
MATRIZ A — Coeficientes de Eficiencia Eléctrica:

Cada elemento aᵢⱼ representa cuánto influye la potencia del subsistema j en la restricción i. Por ejemplo, a₁₂ es la interferencia térmica que genera la CPU sobre el bus del transmisor RF. Los elementos de la diagonal dominante representan la eficiencia propia de cada subsistema.

VECTOR b — Power Budget Disponible [W]:

Cada componente bᵢ es la potencia total que el panel solar puede suministrar bajo la restricción i de la órbita actual. Cambia según el ángulo de incidencia solar, la temperatura de las celdas fotovoltaicas y el estado de las baterías de litio.

// 05 — RIGOR MATEMÁTICO

Algoritmos Implementados

FACTORIZACIÓN LU
A = L·U
x = U⁻¹(L⁻¹b) — método directo, sin iteraciones

Descomposición de Doolittle. Pivoteo parcial para estabilidad. Complejidad O(n³). Sirve como referencia exacta contra la que se mide el error de los métodos iterativos.

JACOBI / GAUSS-SEIDEL
x_i^(k+1) = (b_i − Σaᵢⱼxⱼ) / aᵢᵢ
Jacobi: usa xᵏ · GS: usa xᵏ⁺¹ disponibles

Métodos iterativos clásicos. Convergencia garantizada si A es diagonalmente dominante. GS converge ~2x más rápido que Jacobi en general.

SOR
x_i^(k+1) = (1-ω)x_i^k + ω·x̃_i^(k+1)
ω ∈ (1,2) → sobrerelajación acelerada

Successive Over-Relaxation. El parámetro ω controla la velocidad de convergencia. Óptimo cuando ω > 1. Ajusta el slider en el panel de escenarios.

GRADIENTE CONJUGADO PRECONDICIONADO — Algoritmo 2 (Suñagua, 2020)
Precondicionador: M = diag(A)⁻¹ [trivial, C = diag(A)^(-1/2)]
Resuelve Mzₖ = rₖ en cada iteración
βₖ = r^T_{k-1}·z_{k-1} / r^T_{k-2}·z_{k-2}
pₖ = z_{k-1} + βₖ·p_{k-1}
αₖ = r^T_{k-1}·z_{k-1} / p^T_k·A·pₖ
xₖ = x_{k-1} + αₖ·pₖ
rₖ = r_{k-1} − αₖ·A·pₖ

El GCP transforma el sistema mal condicionado Ax = b en uno equivalente con número de condición κ(M) ≪ κ(A). El precondicionador diagonal M = diag(A)⁻¹ escala cada variable por su propia diagonal, concentrando los autovalores. Esto es fundamental en el Escenario 3 donde los métodos clásicos divergen o convergen extremadamente lento.

κ(A_precond) ≪ κ(A_original)
Convergencia: ‖xₖ−x‖ ≤ 2((√κ−1)/(√κ+1))^k ‖x₀−x‖
Ref: Suñagua, P. (2020). Método de Gradientes Conjugados Precondicionado. Rev. Bol. Mat. UMSA-04.
