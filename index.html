01 — MODELADO DEL SISTEMA

El Satélite como Servidor Orbital

Un satélite de órbita baja (LEO) es esencialmente un servidor de red autónomo a ~550 km de altitud. Su restricción crítica es el Power Budget: la energía eléctrica generada por los paneles solares debe distribuirse entre sus tres subsistemas sin saturación ni apagado. El sistema Ax = b modela este balance, donde cada ecuación representa una restricción térmica o energética del bus eléctrico.

Transmisión de Datos (RF/Láser)

💻

Potencia [W] para el router embarcado. Enruta el tráfico IP de usuarios. Genera calor residual que afecta el subsistema térmico.
⚙️
x₃
Estabilización (Reaction Wheels)
Potencia [W] para los motores de orientación. Mantienen el satélite apuntando correctamente. Consumo variable en maniobras.
02 — SIGNIFICADO DE A Y b

La Matriz de Eficiencia y el Power Budget

Matriz A — Coeficientes de Eficiencia Eléctrica

Cada elemento aᵢⱼ representa cuánto influye la potencia del subsistema j en la restricción i. Por ejemplo, a₁₂ es la interferencia térmica que genera la CPU (x₂) sobre el bus del transmisor RF (x₁). Los elementos de la diagonal dominante representan la eficiencia propia de cada subsistema: su disipación de calor y pérdidas resistivas en función de la temperatura orbital.

Vector b — Power Budget Disponible [W]

Cada componente bᵢ es la potencia que el panel solar puede suministrar bajo la restricción i de la órbita actual. Cambia según: el ángulo de incidencia solar sobre las celdas fotovoltaicas, la temperatura de los paneles (que afecta la eficiencia), el estado de carga de las baterías de litio, y si el satélite está en zona iluminada o en umbra terrestre.

ANÁLISIS DEL CONDICIONAMIENTO

Interpretación Matemática e Ingenieril

Calculando análisis...

κ pequeño → Sistema Robusto

Cuando κ(A) es pequeño (cercano a 1), una perturbación relativa δb/b en el vector de datos produce una perturbación en la solución del mismo orden: δx/x ≤ κ·δb/b. El satélite puede confiar en la solución calculada.

κ grande → Sensibilidad Extrema

Con κ(A) ≫ 1, errores pequeños en la medición de los sensores se amplifican enormemente en la solución. El sistema no puede determinar confiablemente la distribución de potencia: riesgo de daño al hardware.

GCP → Reduce κ Efectivo

El precondicionador M=diag(A)⁻¹ transforma el sistema a Mx̃=b̃ donde κ(M) ≪ κ(A). Los autovalores se concentran alrededor de 1. La convergencia del GCP depende de κ(M), no de κ(A).

MÉTODO DIRECTO

Factorización LU (Doolittle + Pivoteo Parcial)

A = L · U
Lᵢⱼ = aᵢⱼ − Σₖ<ⱼ Lᵢₖ·Uₖⱼ / Uⱼⱼ (i > j)
Uᵢⱼ = aᵢⱼ − Σₖ<ᵢ Lᵢₖ·Uₖⱼ (i ≤ j)
Complejidad: O(n³) · Sin iteraciones · Solución exacta (módulo aritmética finita)

La factorización descompone A en el producto de una matriz triangular inferior L y una superior U. El pivoteo parcial intercambia filas para maximizar la estabilidad numérica. La solución se obtiene por sustitución progresiva (Ly = Pb) y regresiva (Ux = y).

// PSEUDOCÓDIGO LU (Doolittle)
Para k = 0 a n-1: Encontrar maxRow (pivoteo parcial) Intercambiar filas k ↔ maxRow Para i = k+1 a n-1: L[i][k] = U[i][k] / U[k][k] Para j = k a n-1: U[i][j] -= L[i][k] * U[k][j] Sustitución progresiva: Ly = Pb Sustitución regresiva: Ux = y
MÉTODOS ITERATIVOS CLÁSICOS

Jacobi y Gauss-Seidel

Jacobi

x_i^(k+1) = (b_i − Σⱼ≠ᵢ aᵢⱼ xⱼ^(k)) / aᵢᵢ
Dado x⁰ = 0, tol = 10⁻⁶ Para k = 0, 1, 2, ...: Para i = 0 a n-1: s = Σⱼ≠ᵢ a[i][j] * x[j] (usa xᵏ) x_new[i] = (b[i] - s) / a[i][i] err = ‖x_new − x‖∞ x = x_new Si err < tol: CONVERGIÓ → retornar x

Condición suficiente: A diagonalmente dominante estricta.

Gauss-Seidel

x_i^(k+1) = (b_i − Σⱼ<ᵢ aᵢⱼ xⱼ^(k+1) − Σⱼ>ᵢ aᵢⱼ xⱼ^(k)) / aᵢᵢ
Dado x⁰ = 0, tol = 10⁻⁶ Para k = 0, 1, 2, ...: x_old = copia de x Para i = 0 a n-1: s = Σⱼ≠ᵢ a[i][j] * x[j] (usa xᵏ⁺¹ disponibles) x[i] = (b[i] - s) / a[i][i] (in-place) err = ‖x − x_old‖∞ Si err < tol: CONVERGIÓ → retornar x

Converge ~2× más rápido que Jacobi. Misma condición suficiente.

SOBRERELAJACIÓN SUCESIVA

SOR — Successive Over-Relaxation

x̃_i^(k+1) = Gauss-Seidel paso i
x_i^(k+1) = (1 − ω)·x_i^(k) + ω·x̃_i^(k+1)
ω = 1: equivalente a Gauss-Seidel · ω ∈ (1,2): sobrerelajación

El parámetro ω "sobre-corrige" la actualización de Gauss-Seidel. Con ω > 1 se salta hacia adelante en la dirección de la corrección, acelerando la convergencia. El ω óptimo para matrices simétricas definidas positivas depende del radio espectral ρ de la matriz de Jacobi: ω_opt = 2/(1+√(1−ρ²)).

▷ VENTAJA EN EL ESCENARIO 2
En el escenario de estrés, los coeficientes grandes de b hacen que Gauss-Seidel necesite más iteraciones. SOR con ω ajustado converge significativamente más rápido, reduciendo el tiempo de respuesta del sistema de control de potencia del satélite.
// PSEUDOCÓDIGO SOR
Dado x⁰ = 0, ω ∈ (1,2), tol = 10⁻⁶ Para k = 0, 1, 2, ...: x_old = copia de x Para i = 0 a n-1: s = Σⱼ≠ᵢ a[i][j] * x[j] x_GS = (b[i] - s) / a[i][i] x[i] = (1-ω)*x[i] + ω*x_GS err = ‖x − x_old‖∞ Si err < 10⁻⁶: CONVERGIÓ

Gradiente Conjugado Precondicionado — Algoritmo 2

// ALGORITMO 2 — Suñagua, P. Rev. Bol. Mat. UMSA-04 (2020) · Precondicionador: M = diag(A)⁻¹
Precond: M = diag(A)⁻¹ · [trivial: C = diag(A)^(-½)]
Datos: x⁰ (inicial), Ax⁰ ≈ b
r₀ = b − Ax⁰ (residuo inicial)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Mientras rₖ ≠ 0:
  Resolver Mzₖ = rₖ (diagonal: zₖ = M·rₖ)
  Si k=1: p₁ = z₀
  Si k>1: βₖ = rᵀₖ₋₁zₖ₋₁ / rᵀₖ₋₂zₖ₋₂
          pₖ = zₖ₋₁ + βₖ·pₖ₋₁
  αₖ = rᵀₖ₋₁zₖ₋₁ / pᵀₖApₖ
  xₖ = xₖ₋₁ + αₖ·pₖ
  rₖ = rₖ₋₁ − αₖ·A·pₖ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
retornar x = xₖ

¿Por qué es crítico en el Escenario 3?

El GCP transforma el sistema mal condicionado Ax=b en el sistema precondicionado M⁻¹Ax=M⁻¹b donde M=CCᵀ=diag(A). La nueva matriz de coeficientes M⁻¹A tiene número de condición κ(M⁻¹A) ≪ κ(A).

‖xₖ − x‖ ≤ 2·((√κ−1)/(√κ+1))ᵏ·‖x₀−x‖
Tasa de convergencia depende de κ(M⁻¹A), no de κ(A)

Con el precondicionador diagonal, los autovalores de M⁻¹A se concentran cerca de 1, reduciendo drásticamente el cociente (√κ−1)/(√κ+1) y acelerando la convergencia. En el Escenario 3 esto es la diferencia entre converger y diverger.

▷ IMPLEMENTACIÓN EN ESTE PROYECTO
Se implementa exactamente el Algoritmo 2 de Suñagua (2020). El precondicionador M=diag(A)⁻¹ es "trivial" computacionalmente (solo división por el elemento diagonal), pero mejora significativamente κ para matrices con diagonal dominante. Ver js/math.js → solveGCP().
